# バックエンドってなに？

## フロントエンドとバックエンドの役割分担

あなたが普段使っているWebサービスやアプリは、大きく2つの部分に分かれています。

**フロントエンド**は、あなたが実際に見て触る部分です。ボタン、画面のレイアウト、アニメーションなどがフロントエンドの仕事です。

**バックエンド**は、その裏側で動いている部分です。データの処理・保存・計算などを担当しています。フロントエンドから見えることはありませんが、フロントエンドが表示するデータを用意しているのはバックエンドです。

Gopher Slayerで言えば：

- 画面に表示されるゲームのUI → **フロントエンド**
- 勇者のHP・攻撃力の計算、経験値の保存 → **バックエンド**

フロントエンドはあくまで「表示」が仕事で、データの処理はバックエンドに任せます。この役割分担があることで、フロントエンドはデータの見せ方に集中でき、バックエンドはデータの処理に集中できます。

---

## APIとは

### 定義

**API（Application Programming Interface）** とは、異なるソフトウェア同士が会話するための「窓口」です。

フロントエンドはバックエンドのデータが必要な時、直接バックエンドの内部にアクセスすることはできません。その代わりに、バックエンドが用意した「窓口」を通じてデータをやり取りします。この窓口がAPIです。

### Gopher Slayerで考える

Gopher Slayerで「攻撃」ボタンを押した時、裏側では以下のことが起きています。

```
フロントエンド
    ↓ 「攻撃して！」とリクエストを送る
   API（窓口）
    ↓ リクエストを受け取る
バックエンド
    ↓ 攻撃力を計算して、敵のHPを更新する
   API（窓口）
    ↓ 「攻撃成功！敵のHPが10減った」とレスポンスを返す
フロントエンド
    ↓ 結果を画面に表示する
```

フロントエンドはバックエンドの内部実装を知らなくても、APIという窓口を通じてデータをやり取りできます。これがAPIの役割です。

---

## エンドポイントとURLの読み方

APIの窓口には「住所」があります。この住所のことを**エンドポイント**と呼びます。

エンドポイントはURLで表現されます。たとえば：

```
http://localhost:8080/hero
```

このURLは3つの部分に分かれています。

| 部分 | 例 | 意味 |
|---|---|---|
| ベースURL | `http://localhost:8080` | サーバーの住所 |
| パス | `/hero` | どのリソースにアクセスするか |
| クエリパラメータ | `?id=1` | 追加の条件（任意） |

クエリパラメータを使うと、より具体的な情報を指定できます。

```
http://localhost:8080/hero?id=1
```

これは「idが1の勇者の情報をください」という意味になります。

## 実際にAPIを叩いてみよう

### Swaggerとは

**Swagger**はAPIの仕様書であり、ブラウザ上からAPIを実際に叩いて試せるツールです。どんなエンドポイントがあるか、どんなデータが返ってくるかを確認できます。

`docker compose up -d`でGopher Slayerを起動した後、ブラウザで以下のURLにアクセスしてみましょう。

```
http://localhost:8080/swagger/index.html
```

### GET /hero を叩いてみる

1. Swaggerの画面を開く
2. `GET /hero` を探してクリック
3. 「Try it out」ボタンをクリック
4. 「Execute」ボタンをクリック

画面の下に以下のようなデータが返ってきます。

```json
{
  "id": 1,
  "name": "勇者",
  "hp": 100,
  "attack": 15,
  "exp": 0
}
```

これが、バックエンドからフロントエンドに返ってくるデータの形です。

---

## JSONとは

さっきSwaggerで受け取ったデータの形式を**JSON（JavaScript Object Notation）** と言います。

```json
{
  "id": 1,
  "name": "勇者",
  "hp": 100,
  "attack": 15,
  "exp": 0
}
```

JSONは「キーと値のペア」でデータを表現します。

- `"name": "勇者"` → 名前は「勇者」
- `"hp": 100` → HPは100
- `"exp": 0` → 経験値は0

フロントエンドとバックエンドがデータをやり取りする時、このJSON形式を使うのが現在のWebの標準です。GoもJSONの読み書きを標準ライブラリでサポートしています。

---

> **まとめ**
>
> - バックエンドはデータの処理・保存を担当する
> - フロントエンドとバックエンドはAPIという窓口を通じてデータをやり取りする
> - APIの住所をエンドポイントと呼び、URLで表現される
> - データのやり取りにはJSON形式が使われる
