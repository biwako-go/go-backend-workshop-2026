# Dockerってなに？

## 「自分のPCでは動くのに」問題

プログラミングをしていると、こんな場面に出くわすことがあります。

> 「自分のPCでは動くのに、他のPCでは動かない」

原因はほとんどの場合、**環境の差異**です。OSのバージョン、インストールされているライブラリのバージョン、設定ファイルの違いなど、細かい差が積み重なって、「あなたのPCでは動くが、私のPCでは動かない」という状況が生まれます。

複数人が参加するワークショップで全員のPCに同じ環境を手動で用意しようとすると、セットアップだけで半日かかっても不思議ではありません。

これを解決するのが**Docker**です。


## Dockerとは

Dockerは、アプリケーションとその動作に必要な環境をまるごとパッケージ化して、どこでも同じように動かせるツールです。

このパッケージのことを**コンテナ**と呼びます。コンテナの中にはアプリケーションのコードだけでなく、動作に必要なライブラリやOS、設定ファイルまで含まれています。

```
コンテナ
┌─────────────────────────┐
│ アプリケーション          │
│ ライブラリ               │
│ 設定ファイル             │
│ OS（最小限）             │
└─────────────────────────┘
    ↓ どこでも同じように動く
MacでもWindowsでもLinuxでも
```

Dockerを使うことで「自分のPCでは動くのに」問題がほぼ解決します。コンテナさえ配れば、全員が同じ環境でアプリケーションを動かせます。

近年の開発現場では、Dockerはほぼ必須のツールと言っていいでしょう。本番環境へのデプロイ、チーム開発の環境統一、CIの実行環境など、あらゆる場面でDockerが使われています。


## Gopher SlayerとDocker

Gopher Slayerは、フロントエンド・バックエンド・DBの3つのサービスがすべてDockerコンテナとして動いています。

```
docker compose up -d を実行すると...

┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  フロントエンド   │  │  バックエンド     │  │       DB        │
│   コンテナ       │  │   コンテナ       │  │    コンテナ      │
└─────────────────┘  └─────────────────┘  └─────────────────┘
        ↑                    ↑                     ↑
              全部まとめて起動される
```

複数のコンテナをまとめて管理する仕組みを**Docker Compose**と言います。`docker compose up -d` の1コマンドで、今日必要な環境が全員のPCで同じように立ち上がります。


## 覚えるコマンド

今日使うコマンドはこれだけです。

```bash
docker compose up -d
```

- `docker compose up`：`compose.yml` に定義されたコンテナをまとめて起動する
- `-d`：バックグラウンドで起動する（ターミナルを占領しない）
